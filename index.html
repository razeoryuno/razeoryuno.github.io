// Dev Log — In-Memory Social Profile Caching

CHALLENGE

Public player profile data — consumed by every social feature query — was fetched directly from MySQL on each request. As concurrent players browsed friend lists simultaneously, repeated DB reads created a measurable performance bottleneck with no caching layer in place.

SOLUTION

Introduced Memcached to store public player profile data in RAM. Profile entries were populated on write and served directly for all social-path reads, bypassing MySQL for the majority of profile lookups and substantially reducing database load.
